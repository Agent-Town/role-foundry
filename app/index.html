Why this reads stronger for judges

  • Concrete student: the apprentice is implementing Role Foundry itself, not a vague “AI role”.
  • Named teachers: Robin + Neo are the first instructors shaping the student.
  • Honest exam: public curriculum stays visible, hidden holdouts stay sealed.
  • Visible progress: Run 2 clearly improves on Run 1, especially on holdouts.
  • Receipts: judges can inspect changed files, policy snapshot, and transcript evidence.

The loop in plain English

  1. Robin + Neo define the apprentice and what a good Role Foundry slice looks like.
  2. Role Foundry publishes a public curriculum the apprentice can train against.
  3. Separate hidden holdouts stay sealed so the apprentice cannot overfit the exam.
  4. The apprentice ships a slice — copy, UI, score surfaces, or proof artifacts.
  5. The teacher scores the run and turns failure themes into the next curriculum update.

The result is not just a prettier demo. It is a visible teacher → student → hidden-eval → curriculum loop.
