# Spec 007 — Submission Proof + Partner Wiring

## Intent

Package the project so the hackathon submission is easy to trust.

## Requirements

1. `docs/conversation-log.md` must tell a curated, honest build story.
2. Submission metadata must be derivable from repo evidence.
3. Partner integrations must be load-bearing and narrow.
4. The repo must stay easy for judges to inspect.

## Acceptance criteria

- Conversation log is current
- Demo and live mode are both documented honestly
- Partner claims are backed by code or documented stubs with explicit status
- Repo can be walked quickly without oral patch notes

## Done when

A judge can inspect the repo and see an honest, technically coherent build story.
